IEEE two-column conference paper generated from this repository. Every number, table,
and figure traces to repository code, artifacts, or a read-only model/runtime run
(see `figures/make_figures.py` for provenance).

## Build

The paper compiles with a standard IEEEtran toolchain. Two options:

**A. Tectonic (self-contained, recommended — auto-fetches packages, runs BibTeX):**
```bash
tectonic -X compile paper.tex
```

**B. Standard TeX Live (pdflatex + bibtex):**
```bash
pdflatex paper && bibtex paper && pdflatex paper && pdflatex paper
```
`IEEEtran.cls` and `IEEEtran.bst` are bundled, so no separate install is required.

`make` runs option B; `make tectonic` runs option A; `make figures` regenerates the
data-driven figures (requires the repo's Python environment: numpy, matplotlib,
scikit-learn, torch, onnxruntime, pandas).

## Figure provenance
- Data-driven figures (dataset, LORO, per-leg, detection comparison, confusion, ROC,
  PR, reliability, ablation, field issues, latency) are produced by
  `figures/make_figures.py` from `ml/artifacts/*.json` and a **read-only** run of the
  trained model / ONNX runtime on the held-out test split.
- System figures (`architecture_*`, `recovery_state_machine`, timelines) are copied
  from `docs/diagrams/` and `ml/artifacts/plots/`, which are generated by the project's
  diagram and evaluation code.

## Scope note
The detection, calibration, latency, and equivalence results are verified offline under
a leakage-safe protocol; the recovery **logic** is verified by unit tests. The physical
**disentanglement efficacy** of the recovery motions is an engineering assumption pending
real-robot trials, and is labeled as such in the paper (Table on scope of claims). No
recovery success rates are claimed.
